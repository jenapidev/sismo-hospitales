# Centros de Acopio Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A public, map-based directory of relief collection centers (centros de acopio) with have/need inventory, manager + org info, and "Destino de Ayuda", where anyone can register and manage a center after email login.

**Architecture:** Two new public Supabase tables (`acopio_centers`, `acopio_items`). Next.js pages under `/acopio`. Manager auth via the existing Supabase magic-link; ownership (`owner_user_id`) or coordinator gates edits. Map via Leaflet + free OpenStreetMap tiles (client-only). Pure validators are unit-tested; writes go through service-role server actions after an ownership check.

**Tech Stack:** Next.js 15 (App Router), Supabase, Leaflet + react-leaflet, Vitest.

## Global Constraints

- **All center data is public.** No private fields, no proof uploads. Anon reads non-hidden centers + items; writes only via server actions (service role) after auth/ownership checks.
- **Managers authenticate by Supabase magic link** (open to anyone). A center is owned by `owner_user_id`; only the owner or a coordinator edits it/its inventory.
- **Centers are unique by name** (case-insensitive).
- **Map:** Leaflet + OpenStreetMap tiles, no API key. Pin-drop sets lat/lng.
- **Enums (exact):** item `kind` ∈ {`have`, `need`}; reuse `verification_status` enum, using only `unverified` and `coordinator_verified` for centers.
- **Spanish-first** UI, matching existing pages.

`supabase/migrations/0003_acopio.sql`:
```sql
create table acopio_centers (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  address text,
  lat double precision,
  lng double precision,
  aid_destination text,
  manager_name text not null,
  manager_cedula text,
  org_name text,
  org_id text,
  owner_user_id uuid,
  verification_status verification_status not null default 'unverified',
  verified_by uuid,
  verified_at timestamptz,
  hidden boolean not null default false,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);
create unique index acopio_centers_name_uniq on acopio_centers (lower(name));

create table acopio_items (
  id uuid primary key default gen_random_uuid(),
  center_id uuid not null references acopio_centers(id) on delete cascade,
  kind text not null check (kind in ('have','need')),
  name text not null,
  category text,
  quantity numeric,
  unit text,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);
create index acopio_items_center_idx on acopio_items (center_id);

create trigger acopio_centers_set_updated_at
  before update on acopio_centers
  for each row execute function set_updated_at();
create trigger acopio_items_set_updated_at
  before update on acopio_items
  for each row execute function set_updated_at();

alter table acopio_centers enable row level security;
alter table acopio_items enable row level security;
create policy acopio_centers_read on acopio_centers for select using (hidden = false);
create policy acopio_items_read on acopio_items for select using (true);
```

describe("validateCenter", () => {
  it("accepts a complete center and normalizes types", () => {
    const r = validateCenter(center);
    expect(r.ok).toBe(true);
    if (r.ok) {
      expect(r.value.lat).toBe(10.5);
      expect(r.value.lng).toBe(-66.9);
      expect(r.value.managerCedula).toBe("V12345678");

    if (!r.ok) expect(r.errors.lat).toBeTruthy();

  it("rejects a bad kind or missing name", () => {

    expect(validateItem({ kind: "have", name: "" }).ok).toBe(false);
  });
  it("rejects a negative quantity", () => {
    expect(validateItem({ kind: "have", name: "Agua", quantity: "-5" }).ok).toBe(false);

  - `canManageCenter(user, center: { owner_user_id: string | null })` → boolean (owner or coordinator).
- Consumes: existing `isCoordinatorEmail`, `createServerClient`.

- [ ] **Step 1: Add `getUser` + `canManageCenter` to `lib/auth.ts`**
```ts
export async function getUser() {
  const supabase = await createServerClient();
  const { data: { user } } = await supabase.auth.getUser();
  return user ?? null;
}
export function canManageCenter(
  user: { id: string; email?: string | null } | null,
  center: { owner_user_id: string | null }
): boolean {
  if (!user) return false;
  if (center.owner_user_id && center.owner_user_id === user.id) return true;
  return isCoordinatorEmail(user.email, process.env.COORDINATOR_EMAILS ?? "");
}
```

- [ ] **Step 2: Make callback honor `next`** — in `app/auth/callback/route.ts`, read
  `next` from the query, allow only values starting with `/`, redirect there (default `/admin`):
```ts
const next = url.searchParams.get("next");
const dest = next && next.startsWith("/") ? next : "/admin";
return NextResponse.redirect(new URL(dest, url.origin));
```
